# Documentation Access

This folder contains the project documentation in Markdown format.

## Accessing Documentation

In **development environment**, you can view all documentation files through the web interface at:

```
http://localhost:3000/docs
```

## Features

- **Auth Protection**: This documentation is only available on development mode & for students users (@upqroo.edu.mx)
- **Markdown Rendering**: All `.md` files are automatically converted to HTML
- **Tabbed Interface**: Easy navigation between different documentation files
- **Responsive Design**: Works on desktop and mobile devices
- **Search-friendly**: Well-structured content with proper headings

## Available Documentation

The following documentation files are available:

- `API_DOCUMENTATION.md` - Complete API reference
- `CV_UPLOAD_FEATURE.md` - CV upload functionality
- `DOCUMENT_SECURITY.md` - Security implementation details
- `EMAIL_CAMPAIGN_FEATURE.md` - Email campaign system
- `PASSWORD_RECOVERY_FEATURE.md` - Password recovery process
- `PROJECT_OVERVIEW.md` - High-level project overview
- `SURVEY_SYSTEM.md` - Survey functionality documentation
- `SWAGGER_SETUP_SUMMARY.md` - API documentation setup
- `TAB_RELOAD_FIX.md` - Technical fixes and improvements

## Adding New Documentation

To add new documentation:

1. Create a new `.md` file in the `docs/` folder
2. Write your content in standard Markdown format
3. The file will automatically appear in the documentation interface when you refresh the page

## Production Environment

In production, the documentation interface is disabled for security reasons. The documentation files remain in the repository for developer reference.
